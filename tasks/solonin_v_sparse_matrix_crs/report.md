## 1. Введение

Умножение разреженных матриц — одна из ключевых операций в научных вычислениях, машинном обучении и решении систем линейных уравнений. При работе с большими матрицами, большинство элементов которых равны нулю, использование специальных форматов хранения позволяет существенно снизить расход памяти и ускорить вычисления. В данной работе реализовано параллельное умножение разреженных матриц в формате CRS (Compressed Row Storage) с использованием MPI.

## 2. Постановка задачи

**Входные данные:** две разреженные матрицы A (размер ra × ca) и B (размер ca × cb), заданные в формате CRS:
- `vals` — вектор ненулевых значений
- `cols` — вектор индексов столбцов
- `ptr` — вектор указателей на начало каждой строки

**Выходные данные:** матрица C = A × B в формате CRS.

**Ограничения:** элементы типа `double`, считаются ненулевыми если `|x| > 1e-12`.

## 3. Базовый алгоритм (последовательный)

Для каждой строки i матрицы A:
1. Создать временный вектор `tmp` размером cb, заполненный нулями.
2. Для каждого ненулевого элемента `a[i][k]` строки i матрицы A: пройти по строке k матрицы B и прибавить `a[i][k] * b[k][j]` к `tmp[j]`.
3. Собрать ненулевые элементы из `tmp` в результирующую строку матрицы C.

Сложность: O(nnz_A × nnz_B / n) в среднем по строкам.

## 4. Схема распараллеливания

Строки матрицы A распределяются между процессами по схеме round-robin: процесс с рангом r обрабатывает строки `i % world_size == r`.

**Коммуникации:**
1. `BroadcastSizes` — rank 0 рассылает размеры матриц всем процессам через `MPI_Bcast`.
2. `BroadcastB` — rank 0 рассылает всю матрицу B всем процессам через `MPI_Bcast`.
3. `DistributeA` — rank 0 отправляет каждому процессу его строки матрицы A через `MPI_Send/MPI_Recv`.
4. `ComputeLocal` — каждый процесс вычисляет своё подмножество строк результата.
5. `GatherResults` — rank 0 собирает результаты от всех процессов через `MPI_Send/MPI_Recv`, собирает итоговую матрицу C.

## 5. Детали реализации

Структура файлов:
- `common/include/common.hpp` — типы InType, OutType
- `seq/src/ops_seq.cpp` — последовательная реализация
- `mpi/src/ops_mpi.cpp` — MPI реализация
- `tests/functional/main.cpp` — 20 функциональных тестов
- `tests/performance/main.cpp` — тест производительности на матрицах 50000×50000

Важные особенности:
- Матрица B полностью копируется на каждый процесс (broadcast), что оправдано при большом числе строк A.
- Локальные буферы очищаются в начале каждого вызова `RunImpl` для корректной работы в режиме `task_run`.
- Результирующие строки сортируются по индексу столбца.

## 8. Заключение

Реализовано параллельное умножение разреженных матриц в формате CRS с использованием MPI. Схема распараллеливания по строкам матрицы A с рассылкой матрицы B обеспечивает корректную работу при произвольном числе процессов. Основное ограничение — матрица B полностью копируется на каждый процесс, что при очень большой B может быть неэффективно по памяти.
