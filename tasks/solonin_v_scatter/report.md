## 1. Введение

Операция Scatter — одна из базовых коллективных операций в параллельном программировании. В отличие от Broadcast, который рассылает одни и те же данные всем процессам, Scatter распределяет различные части данных между процессами. В данной работе реализована обобщённая версия Scatter с произвольным корневым процессом, использующая только точечные операции Send/Recv.

## 2. Постановка задачи

**Входные данные:**
- Буфер данных на root-процессе (вектор `int`)
- Количество элементов для каждого процесса `send_count`
- Номер root-процесса

**Выходные данные:** каждый процесс получает свою порцию данных размером `send_count`.

**Ограничения:** реализация только через `MPI_Send` и `MPI_Recv`, без использования `MPI_Scatter`.

## 3. Базовый алгоритм (последовательный)

В последовательном режиме root-процесс просто копирует свою порцию данных напрямую в выходной буфер без каких-либо пересылок.

## 4. Схема распараллеливания

Используется схема с деревом процессов:
- Root-процесс отправляет каждому процессу его порцию через `MPI_Send`.
- Каждый не-root процесс получает свои данные через `MPI_Recv`.
- Объём пересылки: `send_count` элементов типа `MPI_INT` на каждый процесс.

**Коммуникационная схема:** звезда от root ко всем остальным процессам.

## 5. Детали реализации

Структура файлов:
- `common/include/common.hpp` — типы InType, OutType
- `seq/src/ops_seq.cpp` — последовательная реализация (копирование порции root)
- `mpi/src/ops_mpi.cpp` — MPI реализация через Send/Recv
- `tests/functional/main.cpp` — функциональные тесты
- `tests/performance/main.cpp` — тест производительности на 4M элементах

Тип передаваемых данных: `MPI_INT`. Root-процесс задаётся параметром и может быть любым процессом в коммуникаторе.
