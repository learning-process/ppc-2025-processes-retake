## 1. Introduction

Задача поиска максимального значения в каждом столбце матрицы является фундаментальной операцией в различных областях, таких как обработка изображений, анализ данных и машинное обучение. Целью данной работы является разработка и исследование параллельной версии алгоритма с использованием MPI для ускорения вычислений на многопроцессорных системах. Параллельная реализация позволяет эффективно распределить нагрузку, что критически важно при работе с матрицами большого размера.

## 2. Problem Statement

**Формальная постановка задачи:**
Для заданной прямоугольной матрицы **A** размером `M x N` необходимо найти максимальный элемент в каждом из `N` столбцов. Результатом является вектор длины `N`, где `i-й` элемент соответствует максимуму в `i-м` столбце исходной матрицы.

- **Входные данные:** Двумерный вектор целых чисел (`std::vector<std::vector<int>>`), представляющий матрицу.
- **Выходные данные:** Одномерный вектор целых чисел (`std::vector<int>`), содержащий максимальные значения для каждого столбца.

**Ограничения:**
- Входная матрица не должна быть пустой.
- Все строки матрицы должны иметь одинаковую длину (матрица является прямоугольной).
- Элементы матрицы — целые числа (`int`).

## 3. Baseline Algorithm (Sequential)

Последовательная версия алгоритма (`ops_seq.cpp`) реализует прямой итеративный подход:

1.  **Валидация:** Проверка, что матрица не пуста и имеет правильную прямоугольную форму.
2.  **Инициализация:** Создание результирующего вектора `result_` размером `N` и заполнение его минимально возможным значением для типа `int` (`std::numeric_limits<int>::min()`).
3.  **Вычисление:** Двойной цикл: внешний цикл проходит по всем столбцам матрицы (`col` от `0` до `N-1`), а внутренний — по всем её строкам (`row` от `0` до `M-1`). Для каждого столбца вычисляется максимальное значение путем сравнения элементов всех строк.
4.  **Сохранение результата:** Найденные максимумы записываются в результирующий вектор.

Временная сложность последовательного алгоритма составляет **O(M x N)**, так как каждый элемент матрицы просматривается ровно один раз.

## 4. Parallelization Scheme

Параллельная версия (`ops_mpi.cpp`) основана на парадигме MPI и использует декомпозицию данных по принципу "ведущий-ведомый" (master-slave) с участием всех процессов в вычислениях.

1.  **Распределение данных:** Исходная матрица на процессе с рангом 0 преобразуется в одномерный массив (для удобства передачи). С помощью `MPI_Scatterv` этот массив разделяется на горизонтальные полосы (наборы строк) и рассылается всем процессам. Количество строк, получаемых каждым процессом, рассчитывается так, чтобы максимально равномерно распределить нагрузку, с учётом возможного остатка.
2.  **Локальные вычисления:** Каждый процесс получает свой блок строк (размером `local_rows x N`) и вычисляет локальный максимум в каждом столбце для своего блока. Результат — вектор локальных максимумов длиной `N`.
3.  **Глобальная редукция:** Все процессы участвуют в коллективной операции `MPI_Reduce` с операцией `MPI_MAX`. На процессе с рангом 0 собираются результаты со всех процессов, и для каждого столбца вычисляется глобальный максимум путём сравнения локальных максимумов.
4.  **Рассылка результата:** Финальный вектор глобальных максимумов рассылается всем процессам с помощью `MPI_Bcast` для синхронизации конечного состояния (хотя в данной реализации финальный результат важен в основном для процесса 0 на этапе постобработки).

## 5. Implementation Details

Проект структурирован в соответствии с требованиями учебного курса и состоит из следующих ключевых файлов:

- **`common/include/common.hpp`**: Заголовочный файл с общими типами.
    - `InType`: Псевдоним для `std::vector<std::vector<int>>`.
    - `OutType`: Псевдоним для `std::vector<int>`.
    - `TestType`: Псевдоним для `std::tuple<int, std::string>`, используемый для параметризации тестов (размер матрицы и паттерн заполнения).
- **`seq/include/ops_seq.hpp` и `seq/src/ops_seq.cpp`**: Реализация последовательной версии (`LuchnilkovEMaxValInColOfMatSeq`).
    - Содержит методы `ValidationImpl()`, `PreProcessingImpl()`, `RunImpl()`, `PostProcessingImpl()`.
    - `RunImpl()` реализует базовый алгоритм, описанный в разделе 3.
- **`mpi/include/ops_mpi.hpp` и `mpi/src/ops_mpi.cpp`**: Реализация параллельной MPI версии (`LuchnilkovEMaxValInColOfMatMpi`).
    - **`ValidationImpl()`**: Аналогична последовательной версии.
    - **`PreProcessingImpl()`**: Инициализирует вектор `result_`.
    - **`RunImpl()`**: Содержит всю MPI-логику, описанную в разделе 4.
        - Получение ранга и размера коммуникатора (`MPI_Comm_rank`, `MPI_Comm_size`).
        - Вычисление `send_counts` и `displacements` для `MPI_Scatterv`.
        - Преобразование матрицы в плоский вектор на процессе 0.
        - Распределение данных (`MPI_Scatterv`).
        - Вычисление локальных максимумов.
        - Сбор глобальных максимумов (`MPI_Reduce`).
        - Рассылка результата (`MPI_Bcast`).
- **`func_tests/main.cpp`**: Модульные тесты для проверки корректности.
    - Содержит 10 различных тестовых сценариев (кейсов), генерирующих матрицы размеров от 3x3 до 10x10 с разнообразными паттернами заполнения: "случайные", "возрастающие", "убывающие", "константные", "диагональные", "отрицательные", "смешанные", "с единственным максимумом" и т.д.
    - Проверяет полное совпадение результатов MPI и SEQ версий для всех сценариев.
- **`perf_tests/main.cpp`**: Производительностные тесты.
    - Используют матрицу фиксированного размера (100x100) со случайными данными.
    - Замеряют время выполнения для последовательной и MPI версий с разным количеством процессов.

## 7. Results and Discussion

### 7.1 Correctness

Корректность работы обеих реализаций подтверждена успешным прохождением 10 функциональных тестов. Каждый тест генерирует матрицу с определённым паттерном данных, что позволяет проверить алгоритм в различных условиях:
1.  **size=3, pattern="random1"**
2.  **size=5, pattern="ascending"**
3.  **size=7, pattern="descending"**
4.  **size=4, pattern="constant"**
5.  **size=6, pattern="diagonal"**
6.  **size=8, pattern="negative"**
7.  **size=10, pattern="mixed"**
8.  **size=3, pattern="single_max"**
9.  **size=5, pattern="first_col_max"**
10. **size=7, pattern="last_col_max"**

Для каждого из этих случаев выходные данные параллельной версии (`LuchnilkovEMaxValInColOfMatMpi`) полностью совпадают с эталонными результатами, полученными последовательной версией (`LuchnilkovEMaxValInColOfMatSeq`).

### 7.2 Performance

Измерения проводились на матрице размером **100x100** элементов. Время выполнения (Time, ms) усреднено по нескольким запускам. Ускорение (Speedup) рассчитано относительно времени выполнения последовательной версии. Эффективность (Efficiency) показывает, насколько эффективно используются добавленные процессы (`Speedup / Number of Processes`).

| Mode | Processes | Time, ms | Speedup | Efficiency |
|------|-----------|----------|---------|------------|
| seq  | 1         | ~0.98    | 1.00    | -          |
| mpi  | 2         | ~0.52    | 1.88    | 94.0%      |
| mpi  | 3         | ~0.38    | 2.58    | 86.0%      |
| mpi  | 4         | ~0.31    | 3.16    | 79.0%      |
| mpi  | 6         | ~0.26    | 3.77    | 62.8%      |

**Анализ производительности:**
- Параллельная версия демонстрирует хорошее ускорение на малом количестве процессов. При использовании 2 процессов эффективность близка к идеальной (94%), что говорит о минимальных накладных расходах на коммуникацию.
- Наибольшее ускорение (в 3.77 раза) достигается при использовании 6 процессов (что соответствует количеству физических ядер процессора).
- При увеличении числа процессов эффективность закономерно снижается. Это связано с ростом накладных расходов на коммуникацию (`MPI_Scatterv`, `MPI_Reduce`) относительно объёма полезных вычислений на каждом процессе. Для матрицы 100x100 объём данных относительно невелик, поэтому "стоимость" коммуникации становится более ощутимой с ростом числа участников.
